Measures how many channels this server can encode at once, using the exact channel pipeline. Pick a demanding item (4K or HDR is ideal), then run: each round encodes a minute of it with one more concurrent stream than the last, until a stream can no longer hold realtime (30 fps). The result is the number to use for Maximum concurrent streams. Each round takes about a minute, the server will be busy while it runs, and all channel streams must be stopped first.
